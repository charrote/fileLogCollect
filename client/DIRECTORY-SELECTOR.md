# Web版客户端目录选择功能说明

## 功能概述

Web版客户端现在支持完整的文件系统目录选择功能，用户可以通过图形界面浏览和选择日志目录。

## 主要功能

### 1. 目录浏览器
- **Windows系统**：首先显示可用的盘符（C:、D:、E:等），选择盘符后浏览该盘符下的目录
- **Unix系统**：从根目录（/）开始浏览
- 支持目录导航（进入子目录、返回上级目录）
- 盘符使用特殊图标和样式标识

### 2. 交互操作
- **单击**：选择目录或盘符
- **双击**：进入目录或选择盘符
- **选择按钮**：确认选择并关闭对话框
- **取消按钮**：关闭对话框

### 3. 目录验证
- 自动验证目录是否存在
- 检查目录可读性
- 提供详细的错误提示

## 使用方法

### 基本操作流程

1. **打开目录选择器**
   - 点击"日志目录"输入框旁边的"浏览..."按钮
   - 系统将打开目录选择对话框

2. **浏览目录**
   - 对话框显示当前目录的子目录列表
   - 双击目录进入子目录
   - 点击"返回上级"返回父目录
   - 单击目录选中（高亮显示）

3. **选择目录**
   - 单击选中目标目录
   - 点击"选择此目录"按钮确认
   - 系统将验证目录并更新输入框

### Windows系统

- 自动检测可用驱动器（C:、D:、E:等）
- 支持网络路径
- 支持UNC路径（\\server\share）

### Unix系统

- 从根目录（/）开始浏览
- 支持挂载点
- 支持符号链接

## API接口

### 1. 获取目录列表

```
GET /api/directories?path=<路径>
```

**参数：**
- `path`：要浏览的目录路径，空字符串表示根目录

**返回：**
```json
[
  {
    "name": "目录名",
    "path": "完整路径",
    "hasChildren": true
  }
]
```

### 2. 验证目录

```
POST /api/validate-directory
Content-Type: application/json

{
  "path": "目录路径"
}
```

**返回：**
```json
{
  "valid": true,
  "path": "目录路径"
}
```

或错误：
```json
{
  "valid": false,
  "error": "错误信息"
}
```

## 界面说明

### 目录选择器组件

1. **面包屑导航**
   - 显示当前完整路径
   - 便于了解当前位置

2. **目录列表**
   - 显示当前目录的子目录
   - 包含返回上级选项
   - 支持滚动浏览

3. **操作按钮**
   - "选择此目录"：确认选择（需要先选择目录）
   - "取消"：关闭对话框

### 视觉反馈

- **选中状态**：蓝色背景，蓝色边框
- **悬停状态**：浅灰色背景
- **目录图标**：📁 文件夹图标
- **箭头指示**：→ 表示可进入，← 表示返回上级

## 错误处理

### 常见错误

1. **目录不存在**
   - 提示："目录不存在"
   - 检查路径是否正确

2. **权限不足**
   - 提示："目录不可读"
   - 检查文件系统权限

3. **路径无效**
   - 提示："指定路径不是目录"
   - 确保输入的是目录而非文件

## 安全考虑

1. **路径限制**
   - 只返回目录，不返回文件
   - 防止访问敏感文件

2. **权限检查**
   - 验证目录可读性
   - 防止选择无权访问的目录

3. **错误处理**
   - 捕获并友好显示错误
   - 不暴露系统详细信息

## 性能优化

1. **懒加载**
   - 只加载当前目录的内容
   - 减少内存占用

2. **异步处理**
   - 使用async/await避免阻塞
   - 提升用户体验

3. **缓存机制**
   - 可扩展添加目录缓存
   - 减少重复访问

## 浏览器兼容性

- Chrome/Edge：完全支持
- Firefox：完全支持
- Safari：完全支持
- IE11：不支持（建议使用现代浏览器）

## 使用示例

### 示例1：选择本地目录

1. 点击"浏览..."按钮
2. 双击"C:"驱动器
3. 双击"Logs"目录
4. 单击"TestLogs"目录
5. 点击"选择此目录"按钮
6. 路径自动填入输入框：`C:\Logs\TestLogs`

### 示例2：选择网络目录

1. 在输入框中手动输入网络路径：`\\server\share\logs`
2. 点击"浏览..."按钮验证路径
3. 如果路径有效，系统会显示目录内容

## 技术实现

### 后端实现

- 使用Node.js的`fs`模块
- `fs.readdir()`读取目录
- `fs.stat()`验证目录
- `fs.access()`检查权限

### 前端实现

- 使用Fetch API调用后端
- 动态生成DOM元素
- 事件监听处理交互
- CSS样式美化界面

## 故障排除

### 问题：目录列表为空

**原因：**
- 目录确实为空
- 权限不足
- 路径错误

**解决：**
- 检查目录内容
- 检查文件权限
- 验证路径正确性

### 问题：无法选择目录

**原因：**
- 未单击选中目录
- 目录验证失败

**解决：**
- 先单击选中目录
- 检查错误提示
- 确认目录存在且可读

### 问题：驱动器不显示

**原因：**
- 驱动器不存在
- 权限不足
- 驱动器未挂载

**解决：**
- 检查磁盘管理器
- 运行管理员权限
- 手动输入路径
