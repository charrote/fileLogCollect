# AT测试采集客户端 - Web版

这是一个轻量级的Web版客户端，专为解决老旧Win7系统（特别是未安装SP1的系统）的兼容性问题而设计。相比Electron版本，Web版客户端具有以下优势：

- **兼容性更好**：支持所有Windows版本，包括Win7无SP1系统
- **体积更小**：无需Electron，部署更简单
- **内网友好**：在内网环境下运行更稳定，依赖更少
- **跨平台访问**：可以从任何设备上的浏览器访问客户端界面

## 技术架构

Web版客户端采用纯Node.js + Express + 前端HTML/CSS/JavaScript的架构，完全避免了Electron的兼容性问题。

- **后端**：Node.js + Express
- **前端**：原生HTML/CSS/JavaScript
- **通信**：RESTful API
- **存储**：本地文件系统

## 安装与使用

### 环境要求

- Node.js >= 14.x
- 现代浏览器（Chrome, Firefox, Edge, Safari等）

### 安装依赖

```bash
cd client
npm install
```

### 启动Web版客户端

```bash
# 启动Web版客户端
npm run start-web

# 或者直接运行
node web-client.js
```

启动后，程序会自动打开默认浏览器并访问客户端界面。如果没有自动打开，请手动访问：

- 本地访问：http://localhost:3002
- 局域网访问：http://[计算机名]:3002

### 配置客户端

1. 在"客户端配置"区域填写必要信息：
   - 设备名称：测试设备的唯一标识
   - 服务端地址：中央服务器的URL（建议使用HTTP而非HTTPS）
   - 日志目录：要监控的日志文件目录
   - 客户端ID：自动生成，通常使用主机名

2. 点击"启动采集"按钮开始监控日志文件

### 使用说明

1. **启动采集**：配置完成后，点击"启动采集"按钮开始监控日志文件
2. **查看状态**：在"运行状态"区域查看客户端运行状态和统计信息
3. **实时日志**：在"实时日志"区域查看详细的运行日志
4. **处理失败项目**：点击"上传失败"数字查看和重新上传失败的项目

## 功能特性

- **实时日志监控**：自动监控指定目录中的日志文件变化
- **智能解析**：自动识别测试开始/结束时间、结果（OK/NG）、统计NG/OK数量
- **失败重试**：支持单个或批量重新上传失败的项目
- **跨平台访问**：可以从任何设备上的浏览器访问客户端界面
- **轻量级部署**：无需安装额外依赖，部署简单

## 与Electron版本的比较

| 特性 | Electron版本 | Web版 |
|------|-------------|-------|
| 兼容性 | 需要Win7 SP1或更高 | 支持所有Windows版本 |
| 体积 | 大（约100MB+） | 小（约10MB） |
| 依赖 | 需要Electron环境 | 只需Node.js |
| 部署 | 需要安装Electron | 简单的npm安装 |
| 访问方式 | 只能在本机访问 | 可从任何设备浏览器访问 |
| 内网支持 | 依赖下载可能有问题 | 更适合内网环境 |

## 常见问题

### 1. 无法访问Web界面

- 检查Node.js是否正确安装
- 检查端口3002是否被占用
- 检查防火墙设置

### 2. 无法连接到服务端

- 确保服务端地址正确
- 尝试使用HTTP而非HTTPS
- 检查网络连接

### 3. 日志文件无法上传

- 检查日志文件路径是否正确
- 检查日志文件格式是否符合要求
- 查看实时日志了解详细错误信息

## 高级配置

### 修改端口

可以通过环境变量修改Web服务器端口：

```bash
# Windows
set WEB_PORT=8080 && node web-client.js

# Linux/Mac
WEB_PORT=8080 node web-client.js
```

### 自定义日志目录

可以通过环境变量设置默认日志目录：

```bash
# Windows
set LOG_DIR=D:\TestLogs && node web-client.js

# Linux/Mac
LOG_DIR=/var/log/tests node web-client.js
```
