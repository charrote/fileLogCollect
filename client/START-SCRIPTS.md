# Web版客户端启动脚本说明

## 概述

Web版客户端提供了两个启动脚本，方便用户快速启动和管理Web客户端服务。

## 启动脚本

### 1. start-web.bat（简单启动）

**特点**：
- ✅ 一键启动，无需配置
- ✅ 自动检查Node.js环境
- ✅ 自动安装依赖包
- ✅ 自动打开浏览器
- ✅ 服务器状态检测
- ✅ 友好的错误提示

**使用方法**：
```bash
# 双击运行
start-web.bat

# 或在命令行中运行
start-web.bat
```

**功能说明**：
1. **环境检查**：自动检查Node.js是否安装
2. **依赖检查**：自动检查并安装依赖包
3. **目录检查**：确保在正确的目录下运行
4. **服务器启动**：启动Web客户端服务
5. **状态检测**：检测服务器是否启动成功
6. **浏览器打开**：自动打开浏览器访问Web界面

**配置选项**：
- 默认端口：3002
- 可通过环境变量修改端口：
  ```bash
  set WEB_PORT=8080
  start-web.bat
  ```

### 2. start-web-advanced.bat（高级启动）

**特点**：
- ✅ 交互式菜单界面
- ✅ 自定义端口配置
- ✅ 调试模式支持
- ✅ 服务状态检查
- ✅ 服务停止功能
- ✅ 依赖管理功能

**使用方法**：
```bash
# 双击运行
start-web-advanced.bat

# 或在命令行中运行
start-web-advanced.bat
```

**菜单选项**：
```
========================================
  请选择操作
========================================

  1. 快速启动（使用默认配置）
  2. 自定义端口启动
  3. 调试模式启动
  4. 检查并安装依赖
  5. 查看运行状态
  6. 停止运行中的服务
  0. 退出
========================================
```

**功能说明**：

1. **快速启动**：使用默认配置（端口3002）快速启动
2. **自定义端口**：输入自定义端口号启动服务
3. **调试模式**：启用调试模式，显示详细日志
4. **安装依赖**：检查并安装依赖包
5. **查看状态**：检查服务运行状态和端口占用情况
6. **停止服务**：停止指定端口的运行服务

## 系统要求

### 必需软件
- **Node.js**：v14.x 或更高版本
  - 下载地址：https://nodejs.org/
  - 推荐下载LTS版本

### 可选软件
- **curl**：用于检测服务器状态（Windows 10+已内置）

## 故障排除

### 问题1：未找到Node.js

**错误信息**：
```
[错误] 未找到Node.js，请先安装Node.js
[提示] 访问 https://nodejs.org/ 下载安装
```

**解决方案**：
1. 访问 https://nodejs.org/
2. 下载并安装LTS版本
3. 重新运行启动脚本

### 问题2：端口被占用

**错误信息**：
```
[错误] 服务器启动失败
[提示] 请检查端口 3002 是否被占用
```

**解决方案**：
1. 使用高级启动脚本查看端口占用情况
2. 使用自定义端口启动
3. 或停止占用端口的服务

### 问题3：依赖安装失败

**错误信息**：
```
[错误] 依赖安装失败
```

**解决方案**：
1. 检查网络连接
2. 尝试手动运行 `npm install`
3. 检查npm配置是否正确

### 问题4：浏览器未自动打开

**可能原因**：
- 默认浏览器未设置
- 系统安全限制

**解决方案**：
- 手动访问 http://localhost:3002
- 检查浏览器设置

## 高级配置

### 环境变量

可以通过设置环境变量来自定义启动行为：

```bash
# 设置自定义端口
set WEB_PORT=8080
start-web.bat

# 设置日志级别（高级模式）
set LOG_LEVEL=debug
start-web-advanced.bat
```

### 批处理参数

虽然脚本主要使用交互式菜单，但也可以通过修改脚本中的默认配置：

```batch
:: 在 start-web.bat 中修改
set WEB_PORT=3002

:: 在 start-web-advanced.bat 中修改
set DEFAULT_PORT=3002
set DEFAULT_LOG_LEVEL=info
```
