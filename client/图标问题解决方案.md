# AT测试采集客户端 - 图标缺失问题解决方案

## 问题描述

在启动Electron客户端时出现以下错误：
```
UnhandledPromiseRejectionWarning: Error: Failed to load image from path '/path/to/client/assets/tray-icon.png'
```

## 问题原因

1. **缺少图标文件**：`assets` 目录下不存在 `tray-icon.png` 和 `icon.png` 文件
2. **Electron应用配置**：GUI应用尝试加载不存在的图标文件
3. **部署不完整**：图标文件可能在部署过程中遗漏

## 解决方案

### 方案一：添加图标文件（推荐）

在 `client/assets/` 目录下放置以下图标文件：

- `icon.png` - 主窗口图标 (建议尺寸: 512x512 或 1024x1024)
- `tray-icon.png` - 托盘图标 (建议尺寸: 16x16, 24x24, 32x32)
- `icon.ico` - Windows图标 (可选)
- `icon.icns` - macOS图标 (可选)

### 方案二：使用默认系统图标（当前实现）

已更新 `main.js` 文件，添加了错误处理逻辑：

1. **图标存在检查**：代码现在会先检查图标文件是否存在
2. **异常处理**：如果图标文件不存在，会使用系统默认图标或继续运行
3. **跨平台兼容**：根据操作系统选择合适的图标格式

## 代码变更

### main.js 更新内容：

1. **窗口图标加载**：添加了文件存在性检查
2. **托盘图标加载**：增加了try-catch错误处理
3. **默认图标路径**：提供了跨平台的默认图标路径

## 启动方式

### GUI模式（带界面）
```bash
# 确保在client目录下
cd client

# 安装Electron（如果没有）
npm install electron --save-dev

# 启动GUI应用
npm start
```

### CLI模式（命令行）
```bash
# 直接运行客户端（无GUI）
node client.js

# 或使用环境变量
SERVER_URL=http://your-server:3000 CLIENT_NAME="My Client" node client.js
```

## 文件结构

```
client/
├── assets/                 # 图标目录
│   ├── icon.png          # 主窗口图标
│   ├── tray-icon.png     # 托盘图标
│   ├── icon.ico          # Windows图标（可选）
│   └── icon.icns         # macOS图标（可选）
├── gui/
│   ├── main.js           # 主进程文件（已修复）
│   ├── renderer.js       # 渲染进程文件
│   └── index.html        # 界面文件
├── client.js             # 客户端主文件
├── config.js             # 配置文件
├── LogUploader.js        # 上传模块
└── LogCollector.js       # 采集模块
```

## 注意事项

1. **内网部署**：确保使用HTTP协议（`http://`）而非HTTPS
2. **图标可选**：图标文件是可选的，应用在没有图标的情况下也能正常运行
3. **错误处理**：当前代码已添加充分的错误处理，避免因图标缺失导致的应用崩溃

## 故障排除

### 如果仍有图标相关错误：
1. 检查assets目录是否存在
2. 确认图标文件格式和路径正确
3. 验证文件权限是否允许读取

### 应用启动检查清单：
- [ ] 服务端正在运行（HTTP协议）
- [ ] assets目录存在（可选）
- [ ] 图标文件存在（可选）
- [ ] 配置文件正确
- [ ] 网络连接正常
