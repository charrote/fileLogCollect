# AT测试采集客户端 - 更新日志功能使用说明

## 更新内容

### 实时日志窗口优化
- **固定高度**：日志窗口高度固定为300px，不会随内容增加而扩展
- **滚动条支持**：当日志内容超出窗口高度时，自动显示滚动条
- **行数限制**：最多保留500行日志，超出部分自动清除
- **视觉美化**：采用深色主题，更好的颜色对比度和边框样式

### 样式改进
- **深色背景**：使用 #1e1e1e 作为日志容器背景色
- **边框样式**：添加左侧3px边框，不同日志级别显示不同颜色
- **字体优化**：使用等宽字体，提高代码和日志的可读性
- **间距调整**：优化行间距和内边距，提升视觉效果

## 如何使用

### 1. 启动客户端
```bash
cd /path/to/client
npm start
```

### 2. 配置客户端
在客户端界面中填写：
- **设备名称**：您的测试设备标识
- **服务端地址**：服务端IP地址和端口
- **日志目录**：要监控的日志文件目录

### 3. 查看实时日志
- 点击"启动采集"按钮开始监控
- 日志将实时显示在"实时日志"区域
- 超过500行的旧日志会自动清除
- 可以滚动查看历史日志

### 4. 测试日志功能
运行测试脚本验证日志功能：
```bash
node test_log_display.js
```

## 日志级别说明

### 信息日志 (info)
- 边框颜色：#3ec9d6（青色）
- 背景：半透明青色
- 用途：一般信息、系统状态

### 成功日志 (success)
- 边框颜色：#2ecc71（绿色）
- 背景：半透明绿色
- 用途：上传成功、操作完成

### 警告日志 (warning)
- 边框颜色：#f39c12（橙色）
- 背景：半透明橙色
- 文字颜色：#f1c40f（黄色）
- 用途：警告信息、需要注意的情况

### 错误日志 (error)
- 边框颜色：#e74c3c（红色）
- 背景：半透明红色
- 文字颜色：#ff6b6b（亮红色）
- 用途：错误信息、上传失败

## 技术特性

### 自动滚动
- 新日志添加时自动滚动到底部
- 保持用户始终看到最新日志

### 性能优化
- 限制最大日志行数（500行）
- 自动清理旧日志，防止内存溢出
- 使用高效的DOM操作

### 响应式设计
- 支持不同屏幕尺寸
- 日志窗口自适应布局

## 故障排除

### 日志不显示
1. 检查客户端是否已启动
2. 确认日志目录路径正确
3. 验证日志文件格式

### 滚动条不工作
1. 检查CSS样式是否正确加载
2. 确认日志内容是否超出窗口高度
3. 检查浏览器控制台是否有错误

### 日志行数过多
- 系统会自动清理超过500行的旧日志
- 如需查看更多历史日志，请检查日志文件

## 最佳实践

1. **定期清理**：虽然系统会自动清理，但建议定期检查日志文件大小
2. **监控性能**：观察客户端运行时的资源占用情况
3. **备份重要日志**：对于重要测试数据，建议定期备份日志文件
4. **合理配置**：根据实际需求调整日志监控目录
