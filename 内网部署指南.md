# AT测试采集系统 - 内网部署配置指南

## 概述

本指南专门针对内网环境部署AT测试采集系统，重点说明如何配置服务端和客户端以使用HTTP协议，避免SSL相关的复杂性和问题。

## 服务端配置

服务端默认使用HTTP协议，无需额外配置SSL证书。

### 默认配置
- 协议：HTTP
- 端口：3000
- 地址：0.0.0.0（接受所有网络接口连接）

### 启动服务端
```bash
cd /path/to/project
npm start
# 或
node server.js
```

### 验证服务端运行
```bash
curl http://localhost:3000/api/status
```

## 客户端配置

客户端配置必须使用HTTP协议，避免SSL握手问题。

### 配置文件修改

编辑 `client/config.js`：

## 初始化数据库
node init-database.js #初始化后端数据库和客户端持久化目录
node init-database.js --client  # 仅初始化客户端持久化目录
node init-database.js --backend # 仅初始化后端数据库

## 迁移客户端持久化数据（如果从旧版本升级）
如果从旧版本升级，需要迁移客户端持久化数据：
```bash
node test/migrate-client-data.js
```

```javascript
// client/config.js
module.exports = {
  // 服务端配置 - 注意使用 http:// 而非 https://
  server: {
    url: process.env.SERVER_URL || 'http://localhost:3000', // 服务端地址，使用HTTP
    uploadEndpoint: '/api/client-log', // 上传日志的API端点
    heartbeatInterval: 30000, // 心跳间隔（毫秒）
  },
  
  // 本地日志监控配置
  collector: {
    logDir: process.env.LOG_DIR || './logs', // 本地日志目录
    filePattern: /\.(log|txt)$/i, // 日志文件模式
    pollInterval: 5000, // 轮询间隔（毫秒）
    batchSize: 10, // 批量上传数量
  },
  
  // 客户端标识
  client: {
    id: process.env.CLIENT_ID || require('os').hostname(), // 客户端ID，默认为主机名
    name: process.env.CLIENT_NAME || 'Default Client', // 客户端名称
    deviceType: process.env.DEVICE_TYPE || 'Test Device', // 设备类型
  }
};
```

### 环境变量配置（推荐用于内网）

使用环境变量启动客户端，指定正确的服务端地址：

```bash
# Linux/Mac
SERVER_URL=http://192.168.x.x:3000 CLIENT_NAME="内网测试设备001" LOG_DIR=/path/to/logs node client.js

# Windows
set SERVER_URL=http://192.168.x.x:3000 && set CLIENT_NAME="内网测试设备001" && set LOG_DIR=C:\path\to\logs && node client.js
```

## 网络配置要点

### 防火墙设置
- 服务端开放端口（默认3000）给客户端访问
- 内网环境下通常不需要复杂的防火墙规则

### IP地址配置
- 服务端使用内网IP地址
- 客户端配置正确的服务端内网IP

### DNS配置（可选）
- 内网DNS可简化IP地址管理
- 可配置主机名解析而非直接使用IP

## 故障排除

### 常见问题

#### 1. 连接被拒绝 (ECONNREFUSED)
- 检查服务端是否正在运行
- 检查IP地址和端口是否正确

#### 2. 连接超时
- 检查网络连通性（ping测试）
- 检查防火墙设置

#### 3. SSL相关错误
- 确认URL使用http://而非https://
- 检查客户端配置文件

### 测试连接
```bash
# 测试服务端是否可达
curl http://<server_ip>:3000/api/status

# 测试客户端配置
node -e "console.log(require('./client/config.js').server.url)"
```

## 安全考虑

虽然使用HTTP协议简化了部署，但在内网环境中仍需注意：

1. **网络隔离**：确保系统部署在安全的内网环境中
2. **访问控制**：限制对服务端端口的访问权限
3. **日志保护**：保护日志数据的安全性和隐私性

## 部署检查清单

- [ ] 服务端已启动且运行正常
- [ ] 服务端URL使用HTTP协议
- [ ] 客户端配置正确的服务端IP地址
- [ ] 网络连通性测试通过
- [ ] 防火墙允许相应端口通信
- [ ] 客户端能够成功上传日志

## 最佳实践

1. **标准化配置**：为所有客户端使用一致的配置模板
2. **监控连接状态**：定期检查客户端与服务端的连接状态
3. **日志审查**：定期审查系统日志以发现潜在问题
4. **备份配置**：备份重要的配置文件以防丢失
